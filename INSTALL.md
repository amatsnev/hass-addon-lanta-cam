# Установка Lanta Camera Proxy

## 1. Добавление репозитория в Home Assistant

1. Откройте **Настройки** → **Надстройки** → **Магазин надстроек**
2. Нажмите **⋮** (три точки) в правом верхнем углу → **Репозитории**
3. Добавьте URL:
   ```
   https://github.com/amatsnev/hass-addon-lanta-cam
   ```
4. Нажмите **Добавить**, затем **Закрыть**

## 2. Установка надстройки

1. В магазине надстроек найдите **Lanta Camera Proxy**
2. Нажмите **Установить**
3. После установки откройте вкладку **Конфигурация**
4. Заполните поля:
   - **Login** — ваш логин от ЛанТа (номер телефона, например `9537092157`)
   - **Password** — ваш пароль от ЛанТа (например `ErG2Gf`)
   - **Refresh interval** — интервал обновления токенов в секундах (по умолчанию `5400` = 1.5 часа)
5. Нажмите **Сохранить**
6. Запустите надстройку

## 3. Проверка работы

Откройте в браузере:
```
http://<IP_ВАШЕГО_HA>:8199/cameras
```

Должен вернуться JSON со списком камер, например:
```json
{
  "cameras": [
    {"id": 5015, "title": "Домофон 1", "stream": "...", "ts": "..."},
    {"id": 5016, "title": "Домофон 2", "stream": "...", "ts": "..."}
  ]
}
```

## 4. Настройка go2rtc для видео в Home Assistant

go2rtc — универсальный стриминг-прокси, который используется Home Assistant для отображения видео в карточках камер.

### 4.1. Установка go2rtc

Если у вас ещё не установлен go2rtc:
1. Установите надстройку **go2rtc** из магазина надстроек (репозиторий `https://github.com/AlexxIT/go2rtc`)
2. Запустите go2rtc

### 4.2. Конфигурация go2rtc

Откройте конфигурацию go2rtc (файл `/config/go2rtc.yaml` или через редактор надстройки) и добавьте потоки камер:

```yaml
streams:
  domofon1: http://172.30.32.1:8199/ts/5015
  domofon2: http://172.30.32.1:8199/ts/5016
  domofon3: http://172.30.32.1:8199/ts/5017
  inner1: http://172.30.32.1:8199/ts/5028
  inner2: http://172.30.32.1:8199/ts/5029
  inner3: http://172.30.32.1:8199/ts/5030
  lift1: http://172.30.32.1:8199/ts/5031
  lift2: http://172.30.32.1:8199/ts/5032
  lift3: http://10.0.11.30:8199/ts/5033
```

**Важно**: IP-адрес `172.30.32.1` — это Docker-шлюз Home Assistant. Если у вас другой IP, замените на свой. Адрес `10.0.11.30` — это IP самого Home Assistant в локальной сети.

> Если потоки не подключаются с `172.30.32.1`, попробуйте заменить на IP вашего Home Assistant (например `10.0.11.30`).

Сохраните и перезапустите go2rtc.

### 4.3. Проверка go2rtc

Откройте в браузере:
```
http://<IP_ВАШЕГО_HA>:1984/api/streams
```

Должен вернуться JSON со списком потоков и статусом подключений.

Проверьте воспроизведение:
```
http://<IP_ВАШЕГО_HA>:1984/stream.html?src=domofon1
```

## 5. Добавление камер в Home Assistant

### Вариант А: WebRTC Card (рекомендуется)

1. Установите интеграцию **WebRTC Camera** через HACS
2. Добавьте карточку WebRTC на дашборд:
   ```yaml
   type: custom:webrtc-camera
   url: rtsp://<IP_ВАШЕГО_HA>:8554/domofon1
   ```

### Вариант Б: Стандартная камера через go2rtc

1. **Настройки** → **Устройства и службы** → **Добавить интеграцию**
2. Найдите **Generic Camera** (или камеру, поддерживающую go2rtc)
3. Укажите поток:
   - RTSP: `rtsp://<IP_ВАШЕГО_HA>:8554/domofon1`
   - Или MSE: `http://<IP_ВАШЕГО_HA>:1984/api/stream.m3u8?src=domofon1`

### Вариант В: Превью без видео (MJPEG)

Если видео не нужно, а достаточно обновляющихся картинок:

1. **Настройки** → **Устройства и службы** → **Добавить интеграцию** → **MJPEG IP Camera**
2. URL картинки: `http://<IP_ВАШЕГО_HA>:8199/preview/5015`

## 6. Логины и пароли — сводка

| Сервис | Логин | Пароль | Где указать |
|--------|-------|--------|-------------|
| ЛанТа (cam.lanta-net.ru) | Номер телефона | Пароль от ЛК | Конфигурация надстройки Lanta Camera Proxy |

## Решение проблем

### Потоки не подключаются в go2rtc
- Проверьте, что надстройка Lanta Camera Proxy запущена и логин/пароль указаны верно
- Откройте `http://<HA_IP>:8199/cameras` — должен вернуть список камер
- Попробуйте заменить `172.30.32.1` на IP вашего Home Assistant

### Видео рывками / зависает
- Это особенность конвертации HLS → MPEG-TS, надстройка использует ffmpeg с флагами минимальной задержки
- Проверьте логи надстройки на ошибки ffmpeg

### Ошибка 403 при просмотре
- Токены привязаны к IP-адресу. Если IP изменился — перезапустите надстройку
- Или отправьте POST запрос на `http://<HA_IP>:8199/refresh` для принудительного обновления
